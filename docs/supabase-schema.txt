Current Supabase schema summary
===============================

Source artifacts
----------------
- `supabase-schema-sql.txt` is the exact SQL export.
- `supabase-schema-kdglchjsogpinspiigyo.png` is the visual diagram.
- This file is the quick plain-text summary.

Tables in the current export
----------------------------
- `sites`
- `jobs`
- `site_contacts`
- `job_contacts`
- `job_assignments`
- `notes`
- `profiles`
- `user_pinned_jobs`
- `user_settings`

Relationships
-------------
- `jobs.site_id -> sites.id`
- `site_contacts.site_id -> sites.id`
- `job_contacts.job_id -> jobs.id`
- `job_assignments.job_id -> jobs.id`
- `job_assignments.user_id -> profiles.id`
- `notes.job_id -> jobs.id`
- `notes.author_user_id -> profiles.id`
- `user_pinned_jobs.user_id -> profiles.id`
- `user_pinned_jobs.job_id -> jobs.id`
- `user_settings.user_id -> profiles.id`

Important corrections from the older docs
-----------------------------------------
- There is no `schedule` table in the current export.
- There are no read views in the current export.
- There is no `storage_directories` table in the current export.
- `profiles.id` is a primary key, but the export does not currently show an FK to `auth.users(id)`.
- `sites` does not currently include city/state/postal/lat/long columns.
- `site_contacts` does not currently include `sort_order` or timestamps.
- `job_assignments` does not currently include status, role, notes, or timestamps.
- `user_pinned_jobs` does not currently include `pinned_at`.
- `user_settings` is present and includes card action side, zoom, schedule view, and calculator toggle fields.
